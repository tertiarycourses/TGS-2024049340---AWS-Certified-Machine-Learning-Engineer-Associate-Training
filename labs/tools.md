## Cost Safety

- Use trainer-provided accounts where available.
- Prefer small sample datasets and short training jobs.
- Stop notebook or Studio apps when not in use.
- Delete endpoints, training jobs, model packages, pipelines, and temporary buckets if instructed.
- Avoid large GPU instances unless explicitly assigned.
- Set budget alerts before running hands-on services.

## Learner Tips

- Separate data preparation, model development, deployment, and monitoring concerns.
- Know when to use SageMaker built-in algorithms, custom frameworks, JumpStart, Bedrock, and AI services.
- Track model metrics, data quality, bias, and cost together.
- Practice explaining why one deployment option fits a use case better than another.
- Keep a weak-topic list for exam revision.
